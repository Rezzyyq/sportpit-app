# Оцінка проєкту sportpit-app

## Загальна оцінка
- **Поточний стан:** робочий MVP з фронтендом на React + Vite та бекендом на Express/MongoDB.
- **Оцінка:** **6.5/10** (гарна база, але є архітектурні та якісні ризики перед масштабуванням).

## Сильні сторони
- Чітко розділено фронтенд і бекенд за папками (`src/` і `backend/`).
- Є базовий CRUD для товарів через REST API.
- Виробнича збірка фронтенду успішно проходить (`vite build`).
- Є seed-скрипти для початкового наповнення даних.

## Основні ризики
1. **Підтримуваність UI-логіки**  
   `src/Content.jsx` містить занадто багато відповідальностей (завантаження API, стан форми, пошук, відображення, список відправок), що ускладнює розвиток.
2. **Дублювання та неузгодженість роутів**  
   Є обидва файли `backend/routes/productRoutes.js` і `backend/routes/products.jsx`, що може плутати команду.
3. **Неповна документація проєкту**  
   `README.md` залишився шаблонним від Vite і не описує запуск бекенду/БД.
4. **Якість коду / лінтинг**  
   До виправлень були проблеми ESLint (syntax error та node globals), що може ламати CI.
5. **Безпека та конфігурація**  
   Немає явної валідації вхідних даних і централізованого error-handling middleware.

## Рекомендації (пріоритет)
### Високий пріоритет
- Розбити `Content.jsx` на модулі (`ProductsView`, `ShipmentView`, `SettingsView`, кастомні hooks для API).
- Видалити зайві/дублюючі роут-файли і залишити один канонічний варіант.
- Оновити `README.md`: запуск frontend/backend, змінні оточення, приклад `.env`, seed-команди.

### Середній пріоритет
- Додати валідацію запитів (наприклад, `zod`/`joi`) і централізований error handler на бекенді.
- Додати мінімальні тести:
  - backend: smoke-тести API,
  - frontend: тести рендеру/основних сценаріїв.
- Винести API URL у конфіг (`VITE_API_URL`) замість хардкоду.

### Низький пріоритет
- Уніфікувати стилізацію (зменшити inline-стилі, підтримати дизайн-систему).
- Додати простий CI pipeline (lint + build + tests).

## Що вже покращено в цій ревізії
- Виправлено синтаксичну помилку у `backend/routes/products.jsx`.
- Додано Node globals в `eslint.config.js` для backend-файлів.
- Прибрано невикористані `catch`-змінні у `backend/server.js`.
- Після змін проєкт проходить `npm run lint` та `npm run build`.
