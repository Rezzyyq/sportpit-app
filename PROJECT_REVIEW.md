# Оцінка проєкту sportpit-app

## Загальна оцінка

Поточний стан: робочий MVP із модульним frontend, базовим CRUD на backend, централізованою обробкою помилок і smoke-перевірками.

Оцінка після першого проходу: 7/10.

## Що вже добре

- Frontend розділений на `views/`, `hooks/` і допоміжні утиліти.
- Є окремий `useProducts` для API-операцій з товарами.
- Backend має окремий роутинг для продуктів і централізовані middleware помилок.
- Є базова валідація payload для `POST` і `PUT`.
- Є healthcheck `GET /`.
- Є backend smoke-тести для основних API-сценаріїв.
- Є frontend smoke-тести для структури і ключового wiring.
- Production build проходить.

## Поточні ризики

1. Frontend-тести поки що перевіряють структуру, а не реальну поведінку компонентів у браузері.
2. UI ще потребує полірування: форми, таблиці, empty/error states і мобільна адаптація.
3. Частину UX уже винесено з `prompt/confirm`, але можна зробити повноцінні модалки або toast-повідомлення.
4. Варто узгодити root/backend залежності й запуск так, щоб новому розробнику було простіше стартувати.

## Наступні задачі

- Довести UI відправки та видалення до фінального вигляду.
- Додати поведінкові frontend-тести для CRUD/відправки.
- Покращити адаптивність каталогу і форми.
- Додати нормальні повідомлення про успішні та помилкові дії.
